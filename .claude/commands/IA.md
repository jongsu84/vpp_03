서비스의 IA(Information Architecture, 정보구조도)를 작성합니다.

서비스 개요와 주요 기능을 기반으로 아래 구조로 IA를 작성해주세요.

## IA 작성 구조

### 1. 문서 개요
- 서비스명
- 작성일 / 버전 / 작성자
- 대상 플랫폼 (웹 / 모바일 앱 / 둘 다)

### 2. IA 계층 구조

트리 형태로 전체 구조를 표현합니다:

```
[서비스명]
├── 1. (대메뉴)
│   ├── 1.1 (중메뉴)
│   │   ├── 1.1.1 (소메뉴/화면)
│   │   └── 1.1.2 (소메뉴/화면)
│   └── 1.2 (중메뉴)
├── 2. (대메뉴)
│   └── ...
└── (공통)
    ├── 헤더
    ├── 푸터
    └── 공통 팝업/모달
```

### 3. 화면 목록 (Screen Inventory)
| 화면 ID | 화면명 | 뎁스 | 상위 화면 | URL/경로 | 설명 | 비고 |
|--------|-------|-----|---------|---------|------|------|

화면 ID 규칙: 대메뉴번호-중메뉴번호-화면번호 (예: M01-01-01)

### 4. 내비게이션 구조
- **GNB (Global Navigation Bar)**: 전체 서비스 공통 상단 메뉴
- **LNB (Local Navigation Bar)**: 섹션별 좌측 메뉴
- **BNB (Bottom Navigation Bar)**: 모바일 하단 탭 (해당 시)
- **Breadcrumb**: 경로 표시 방식

### 5. 사용자 유형별 접근 구조
| 사용자 유형 | 접근 가능 메뉴 | 제한 메뉴 |
|-----------|------------|---------|

### 6. 화면 전환 흐름 (Key Flow)
주요 사용자 시나리오별 핵심 화면 흐름을 기술합니다.

예:
- 회원가입 플로우: 랜딩 → 회원가입 → 이메일 인증 → 프로필 설정 → 메인
- 핵심 기능 플로우: ...

작성 시 유의사항:
- 메뉴 뎁스는 3단계 이내를 권장합니다
- 각 화면은 고유 ID를 부여합니다
- 팝업/모달도 화면으로 관리합니다
- 사용자 권한별 접근 범위를 명확히 구분합니다
